## Development Philosophy

> Clean workflows. Clear documentation. Reproducible results.

- **Reproducibility first**: every project should explain how to run it.
- **Documentation matters**: code should be understandable after the project is finished.
- **Visualize results clearly**: plots should help explain the analysis, not just decorate it.
- **Use the right tool for the problem**: Python, R, SQL, and `C++/ROOT` each have a place.
- **Keep public repositories polished**: public GitHub should look like a portfolio, not a storage folder.
